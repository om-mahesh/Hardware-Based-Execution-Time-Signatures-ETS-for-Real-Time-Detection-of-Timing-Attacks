# IEEE Paper Submission Checklist

## Pre-Submission Checklist

### Content Completeness
- [x] **Abstract** - 150-250 words summarizing contribution
- [x] **Introduction** - Motivation, problem, solution, contributions
- [x] **Related Work** - Comprehensive survey of 15+ papers
- [x] **Methodology** - Detailed experimental design
- [x] **Implementation** - Hardware and software details
- [x] **Results** - Quantitative data with statistical analysis
- [x] **Discussion** - Interpretation and limitations
- [x] **Conclusion** - Summary and future work
- [x] **References** - 13 citations (minimum 12 required)

### Figures and Tables
- [ ] **Figure 1**: ETS Architecture diagram (high-quality)
- [ ] **Figure 2**: Timing comparison plot
- [ ] **Figure 3**: Detection results bar chart
- [ ] **Figure 4**: Tolerance trade-off curve
- [x] **Table 1**: Memory map
- [x] **Table 2**: Resource utilization
- [x] **Table 3**: Anomaly detection results
- [x] **Table 4**: Tolerance configuration impact
- [x] **Table 5**: Performance overhead
- [x] **Table 6**: Power consumption

**Figure Requirements:**
- [ ] All figures at 300+ DPI
- [ ] Vector graphics where possible (PDF/EPS)
- [ ] Readable text (12pt minimum in figure)
- [ ] Color-blind friendly palettes
- [ ] High contrast for B&W printing
- [ ] Captions explain key insights

### Writing Quality
- [ ] **Grammar check** - Run through Grammarly or similar
- [ ] **Spell check** - No typos in technical terms
- [ ] **Consistency** - Terminology used uniformly
- [ ] **Passive voice** - Minimize passive constructions
- [ ] **Sentence length** - Average <25 words
- [ ] **Paragraph length** - 4-8 sentences each
- [ ] **Acronyms** - Defined on first use
- [ ] **Numbers** - Consistent formatting (e.g., "16 KB" not "16KB")

### Technical Accuracy
- [ ] **All claims referenced** - Every claim has citation or data
- [ ] **Statistical significance** - p-values reported where appropriate
- [ ] **Error bars** - Standard deviation or confidence intervals shown
- [ ] **Reproducibility** - Enough detail to replicate experiments
- [ ] **Open science** - Code/data availability statement
- [ ] **Hardware specs** - Complete component specifications
- [ ] **Software versions** - All tool versions documented

### IEEE Formatting
- [ ] **Template** - Using current IEEE conference template
- [ ] **Column format** - Two-column layout
- [ ] **Font** - Times New Roman, 10pt
- [ ] **Page limit** - Within venue's page limit (typically 8-12)
- [ ] **Margins** - IEEE standard margins maintained
- [ ] **Headers/footers** - Removed (added by publisher)
- [ ] **Page numbers** - Removed (added by publisher)

### References
- [ ] **Bibliography style** - IEEE format
- [ ] **DOI/URLs** - Included where available
- [ ] **Completeness** - All cited works in bibliography
- [ ] **No orphans** - All bibliography entries cited in text
- [ ] **Venue info** - Conference/journal names correct
- [ ] **Year accuracy** - Publication years verified
- [ ] **Author names** - Correct spelling and order

### Author Information
- [ ] **Names** - All authors listed with correct affiliations
- [ ] **Affiliations** - Institutions, departments, addresses
- [ ] **Email** - Corresponding author email
- [ ] **ORCID** - ORCID iDs for all authors (if required)
- [ ] **Conflicts of interest** - Disclosed if any
- [ ] **Author contributions** - Documented (if required)

### Ethics and Compliance
- [ ] **IRB approval** - If human subjects involved (N/A for this work)
- [ ] **Informed consent** - If required (N/A)
- [ ] **Data privacy** - No sensitive data disclosed
- [ ] **Dual submission** - Not submitted elsewhere simultaneously
- [ ] **Plagiarism check** - Passed Turnitin/iThenticate
- [ ] **Self-plagiarism** - New content, not recycled from prior work
- [ ] **Funding acknowledgment** - Grant numbers listed

## Submission Package

### Required Files
- [ ] **PDF of paper** - Main submission file
- [ ] **Source files** - LaTeX source (if requested)
- [ ] **Figures** - All figures as separate files
- [ ] **Supplementary materials** - If allowed by venue

### Optional Materials
- [ ] **Cover letter** - Explaining significance
- [ ] **Suggested reviewers** - 3-5 experts in field
- [ ] **Conflicts of interest** - Reviewers to exclude
- [ ] **Highlights** - Bullet points of contributions

## Artifact Evaluation (If Applicable)

### Code Repository
- [ ] **Public repository** - GitHub/GitLab with open license
- [ ] **README** - Installation and usage instructions
- [ ] **Dependencies** - All requirements documented
- [ ] **Build scripts** - Automated compilation
- [ ] **Tests** - Unit tests and integration tests
- [ ] **Examples** - Sample programs demonstrating features
- [ ] **Documentation** - API reference and user guide

### Hardware Artifacts
- [ ] **RTL source** - All Verilog/VHDL files
- [ ] **Synthesis scripts** - Vivado/Quartus project files
- [ ] **Constraints** - Timing and pin assignments
- [ ] **Bitstream** - Precompiled for target board
- [ ] **Bill of materials** - Hardware components list
- [ ] **Assembly instructions** - If custom hardware

### Experimental Data
- [ ] **Raw data** - All experimental measurements
- [ ] **Analysis scripts** - Python/R scripts for processing
- [ ] **Generated figures** - Scripts to reproduce figures
- [ ] **Statistical tests** - Code for significance testing
- [ ] **README** - Explaining data format and usage

### Docker Container
- [ ] **Dockerfile** - Complete environment specification
- [ ] **Docker Hub** - Published container image
- [ ] **Run script** - One-command reproduction
- [ ] **Expected output** - Sample results for verification

## Venue-Specific Requirements

### IEEE S&P (Oakland)
- [ ] Page limit: 13 pages (2-column format)
- [ ] Anonymization: Double-blind review
- [ ] Artifact evaluation: Optional but encouraged
- [ ] Ethics: Statement required if human subjects

### USENIX Security
- [ ] Page limit: 16 pages (2-column format)
- [ ] Anonymization: Double-blind review
- [ ] Artifact appendix: Up to 2 additional pages
- [ ] Open access: Will be freely available

### ACM CCS
- [ ] Page limit: 12 pages (2-column format)
- [ ] Anonymization: Double-blind review
- [ ] Reproducibility: Highly valued
- [ ] ACM templates: Use latest version

### IEEE HOST
- [ ] Page limit: 8 pages (2-column format)
- [ ] Anonymization: Single-blind review
- [ ] Hardware focus: Emphasize FPGA implementation
- [ ] Industrial relevance: Highlight practical deployment

### CHES
- [ ] Page limit: 25 pages (1-column LNCS format)
- [ ] Anonymization: Double-blind review
- [ ] Cryptographic detail: In-depth algorithm analysis
- [ ] Side-channel focus: Detailed attack/defense analysis

### Internal Review
- [ ] **Co-author approval** - All authors reviewed and approved
- [ ] **Advisor sign-off** - PI/advisor approved submission
- [ ] **Lab review** - Group feedback incorporated
- [ ] **Mock review** - Anticipated reviewer concerns addressed

### External Review
- [ ] **Colleague feedback** - External expert reviewed
- [ ] **Writing center** - Professional editing (if available)
- [ ] **Technical review** - Domain expert validated claims

## Final Checks

### Before Clicking Submit
- [ ] **Correct venue** - Submitting to intended conference/journal
- [ ] **Deadline** - Submitting before deadline (with time buffer)
- [ ] **File size** - Within submission system limits
- [ ] **PDF compliance** - PDF/A or venue-required format
- [ ] **Metadata** - Title, abstract, keywords entered correctly
- [ ] **Contact info** - Corresponding author email verified
- [ ] **Submission type** - Regular paper, short paper, etc.
- [ ] **Backups** - All files backed up before submission

### After Submission
- [ ] **Confirmation email** - Received submission confirmation
- [ ] **Paper ID** - Noted for future reference
- [ ] **Calendar reminder** - Notification decision date saved
- [ ] **Author response** - Prepared to address reviews (if allowed)
